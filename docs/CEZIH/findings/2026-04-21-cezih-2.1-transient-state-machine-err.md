# TC16 `ERR_HEALTH_ISSUE_2000` — CEZIH test-env transient, retry resolves

## Finding

`ERR_HEALTH_ISSUE_2000` ("Unexpected error occurred while executing state
machine perform.") on `$process-message` event 2.1 is a **CEZIH test-env
transient**, not a payload defect. Retrying the same signed bundle within
~60 s returns HTTP 200.

Counts in the same family as `ERR_DOCTRANSVAL_1000` (ITI-65 replace/storno
transient against CEZIH's internal `localhost:8080` FHIR server — see
`2026-04-20-cezih-test-env-fhir-server-down.md`).

## Evidence

2026-04-21 E2E on prod `app.hmdigital.hr` against real CEZIH test env
(`pvsek.cezih.hr`), patient GORAN PACPRIVATNICI19 (MBO 999990260), ICD J00,
Certilia mobile signing:

| Time (UTC) | verificationStatus | meta.profile | asserter | Result |
|-----------:|--------------------|--------------|----------|--------|
| 07:45 | confirmed | — | present | **400 ERR_HEALTH_ISSUE_2000** |
| 07:49 | confirmed | + bundle + header | present | **400 ERR_HEALTH_ISSUE_2000** |
| 07:51 | confirmed | + bundle + header | dropped | **400 ERR_HEALTH_ISSUE_2000** |
| 07:53 | unconfirmed | + bundle + header | present | **400 ERR_HEALTH_ISSUE_2000** |
| 07:56 | confirmed | + bundle + header | present | **200 — case `cmo8bxbcu023ehb85nzsggu23` created** |

The final successful call is byte-identical to the prior three failing calls
(same MBO/ICD/verificationStatus/asserter/profile annotations; only bundle UUIDs
and timestamps differ). **No payload change → HTTP 200.** Transient confirmed.

## Impact

- TC16 (Create case, Croatian MBO, Certilia mobile) = GREEN.
- No code fix required; existing payload is accepted by CEZIH state machine
  once the transient resolves.
- Before the exam or a demo, if `ERR_HEALTH_ISSUE_2000` hits: **retry once
  within ~60 s**. Don't interpret a single failure as a code bug.
- If the examiner sees the error: show them the CEZIH-side display text
  ("Unexpected error occurred while executing state machine perform.") which
  is an internal exception, not a validation rejection (no specific
  `ERR_*` validation code).
